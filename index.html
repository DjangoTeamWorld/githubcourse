git init

creates repo in our directory on computer

git config --global user.name "your name" and git config --global user.email your email

sets up your name and email address

git status

our repository informations. Branch master or main - this is default main branch. Repository includes three areas. Working Directory - The directory in which we work, we add files. We have two types of files and directories, track and untrack Truck - Files seen by git. Stading Area - Files go to commit from there. Repository - Files after commit go to the repository.

git add . or git add -A

Adds all files for trucking.

git commit -m "This is first commit . File index.html"

git commit -am"fast way on commit and add files to repo

Adds files for repository. After each modification of the file, repeat add, commit.

git log

History on a specific branch.

git branch

Shows what we are currently on branch.

Rule, we do not work on branch master. There must always be code working correctly on branch master. Only after verifying the code, we connect the new code to branch master

git checkout -b "name new branch"

Creates a new branch - Now we are working on a copy of the repository. We can extend the code. After finishing work, switch to master.

git checkout master

Switch to branch master.

git push

Updates the remote branch with local commits.

git push -u origin 'your branch'

This code will create your branch in your remote repository and will push it.

git push origin master

pushes to the master repository

git merge here name branch.

When we are on master, we merge the changes from the branch with the main branch

git remote add origin here adres your github repository.

git clone here adres repository

Copies the repository from github.

git pull

gets changes from github if the other person added changes.
